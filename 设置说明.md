# OlmoEarth Inference 使用说明

## 快速开始

### 1. 下载模型权重

从HuggingFace下载OlmoEarth-v1-Base：
```bash
# https://huggingface.co/allenai/OlmoEarth-v1-Base
```

将`weights.pth`放入`params/`目录。

### 2. 安装依赖

```bash
pip install torch numpy rasterio einops
```

### 3. 运行推理

```bash
cd olmoearth_inference
python olmoearth_inference_jp2_simple.py --jp2_dir /path/to/sentinel2_safe
```

## 完整数据流程详解

本节详细说明从原始Sentinel-2数据到模型输出的完整数据处理流程。

### 1. 数据文件夹结构

Sentinel-2 L2A产品采用SAFE格式，结构如下：

```
S2A_MSIL2A_20250722_22_utm32n/
└── S2A_MSIL2A_20250722_22_utm32n.SAFE/
    └── GRANULE/
        └── L2A_T32UMQ_N0510_20250722_123456/
            └── IMG_DATA/
                └── R10m/
                    ├── T32UMQ_20250722_22_B02_10m.jp2   # Blue
                    ├── T32UMQ_20250722_22_B03_10m.jp2   # Green
                    ├── T32UMQ_20250722_22_B04_10m.jp2   # Red
                    ├── T32UMQ_20250722_22_B08_10m.jp2   # NIR
                    └── ...
```

模型需要读取12个波段（见 `data/constants.py` 第246-259行定义的 `Modality.SENTINEL2_L2A`）：

```python
# data/constants.py 第246-259行
SENTINEL2_L2A = ModalitySpec(
    name="sentinel2_l2a",
    tile_resolution_factor=16,
    band_sets=[
        # 10m分辨率波段 (BandSet 0)
        BandSet(["B02", "B03", "B04", "B08"], 16),
        # 20m分辨率波段 (BandSet 1)
        BandSet(["B05", "B06", "B07", "B8A", "B11", "B12"], 32),
        # 60m分辨率波段 (BandSet 2)
        BandSet(["B01", "B09"], 64),
    ],
    is_multitemporal=True,
    ignore_when_parsing=False,
)
```

### 2. 读取JP2文件并转换为NumPy数组

**源代码：** `olmoearth_inference_jp2_simple.py` 第29-79行

```python
def read_sentinel2_jp2_files(jp2_dir: str, target_size: int = 512) -> np.ndarray | None:
    """读取Sentinel-2 L2A的JP2文件
    
    Returns:
        image: np.ndarray with shape [H, W, num_bands] - Channel-Last格式
    """
    band_order = Modality.SENTINEL2_L2A.band_order  # ['B02', 'B03', 'B04', 'B05', 'B06', 'B07', 'B08', 'B8A', 'B09', 'B11', 'B12', 'B01']
    
    fnames = []
    for band_name in band_order:
        pattern = f"{jp2_dir}/*.SAFE/GRANULE/*/IMG_DATA/*/*_{band_name}_*.jp2"
        matches = glob.glob(pattern)
        if matches:
            fnames.append(matches[0])
    
    # 使用rasterio读取所有波段
    images = []
    for fname in fnames:
        with rasterio.open(fname) as src:
            with rasterio.vrt.WarpedVRT(src, ..., width=target_size, height=target_size) as vrt:
                images.append(vrt.read(1))
    
    # [H, W, num_bands=12] - Channel-Last格式
    image = np.stack(images, axis=-1)
    return image
```

**输出：** `np.ndarray` shape `[H, W, 12]` (Channel-Last格式)

### 3. 归一化 (Normalization)

**源代码：** `data/normalize.py` 第39-132行

OlmoEarth支持两种归一化策略，通过 `Strategy` 枚举选择：

```python
class Strategy(Enum):
    PREDEFINED = "predefined"  # 使用预定义的min/max值
    COMPUTED = "computed"      # 使用预计算的mean/std值
```

#### 策略1：PREDEFINED（使用预定义min/max）

**参数来源：** `data/norm_configs/predefined.json`

**归一化公式**（`data/normalize.py` 第78-95行）：
```
normalized = (data - min_val) / (max_val - min_val)
```

这种方式直接使用预定义的最小值和最大值将数据线性缩放到[0,1]区间。

#### 策略2：COMPUTED（使用mean/std计算）

**参数来源：** `data/norm_configs/computed.json`

**归一化公式**（`data/normalize.py` 第97-116行）：
```
min_val = mean - 2 * std
max_val = mean + 2 * std
normalized = (data - min_val) / (max_val - min_val)
```

这种方式基于预计算的均值和标准差，将数据标准化到覆盖约95%数据分布的范围内。

以B02波段为例（`data/norm_configs/computed.json`）：
- mean = 1188.94
- std = 1859.19
- min_val = 1188.94 - 2×1859.19 = -2529.44
- max_val = 1188.94 + 2×1859.19 = 4907.32

#### 推理脚本中的使用

**源代码：** `olmoearth_inference_jp2_simple.py` 第82-102行

```python
def normalize_image(image: np.ndarray) -> np.ndarray:
    """归一化图像"""
    # 使用COMPUTED策略
    normalizer = Normalizer(strategy=Strategy.COMPUTED)
    modality = Modality.SENTINEL2_L2A
    
    # 添加batch和time维度: [H, W, C] -> [1, H, W, 1, C]
    image_with_batch = image[np.newaxis, :, :, np.newaxis, :]
    normalized = normalizer.normalize(modality, image_with_batch.astype(np.float32))
    
    # 去掉batch和time维度: [1, H, W, 1, C] -> [H, W, C]
    normalized = normalized[0, :, :, 0, :]
    return normalized
```

> ⚠️ **注意：** 推理时没有数据增强（Data Augmentation）。数据增强仅在训练时使用（如 `nn/flexi_vit.py` 第333-344行的 `band_dropout` 仅在 `self.training=True` 时生效）。

### 4. 准备输入张量 (Tensor Preparation)

**源代码：** `olmoearth_inference_jp2_simple.py` 第105-134行

```python
def prepare_input(normalized_image: np.ndarray, crop_size: int = 128) -> tuple[torch.Tensor, torch.Tensor]:
    """准备模型输入"""
    h, w = normalized_image.shape[:2]
    
    # 中心裁剪
    start_h = (h - crop_size) // 2
    start_w = (w - crop_size) // 2
    cropped = normalized_image[start_h:start_h + crop_size, start_w:start_w + crop_size, :]
    
    # 转换为tensor并添加batch和time维度: [H, W, C] -> [1, H, W, 1, C]
    image_tensor = torch.tensor(cropped, dtype=torch.float32).unsqueeze(0).unsqueeze(3)
    
    # 创建mask: [B, H, W, T, S] - S=3是band sets数量
    from datatypes import MaskValue
    B, H, W, T, C = image_tensor.shape
    S = 3  # band sets数量
    mask = torch.full((B, H, W, T, S), MaskValue.ONLINE_ENCODER.value, dtype=torch.int64)
    
    return image_tensor, mask
```

**关键概念 - BandSet 转换：**

BandSet是理解数据流的核心概念。在 `data/constants.py` 第48-82行定义：

```python
@dataclass(frozen=True)
class BandSet:
    """A group of bands that is stored at the same resolution.

    Many modalities only have one band set, but some have different bands at different
    resolutions.
    """
    bands: list[str]                      # 波段名称列表
    resolution_factor: int               # 分辨率因子
```

Sentinel-2 L2A有3个BandSet：
| BandSet索引 | 波段 | 原始分辨率 | 存储分辨率 | 通道数 |
|-------------|------|-----------|-----------|--------|
| 0 | B02, B03, B04, B08 | 10m | 10m | 4 |
| 1 | B05, B06, B07, B8A, B11, B12 | 20m | 20m | 6 |
| 2 | B01, B09 | 60m | 40m | 2 |

**BandSet如何改变Tensor Shape：**

在 `nn/flexi_vit.py` 第294-353行的 `MultiModalPatchEmbeddings.apply_embedding_to_modality` 方法中：

```python
# nn/flexi_vit.py 第309-353行
for idx in range(num_band_sets):  # 遍历每个bandset
    buffer_name = self._get_buffer_name(modality, idx)
    # 使用buffer索引从原始数据中提取对应bandset的波段
    inp_data = torch.index_select(modality_data, -1, getattr(self, buffer_name))
    
    embedding_module = self.per_modality_embeddings[modality][
        self._get_embedding_module_name(modality, idx)
    ]
    patchified_data = embedding_module(inp_data, **modality_specific_kwargs)
    
    modality_tokens.append(patchified_data)  # 收集每个bandset的token
    modality_masks.append(token_mask)

# 最后stack: [B, H', W', T, S, D]
return torch.stack(modality_tokens, dim=-2), torch.stack(modality_masks, dim=-1)
```

**Shape转换流程：**
```
输入: [B, H, W, T, C=12] (12个波段连续排列)
         ↓
    ┌───────────────────────────────────────┐
    │  BandSet索引选择 (通过buffer)          │
    │  BandSet 0: 通道[0,1,2,7] → 4通道     │
    │  BandSet 1: 通道[3,4,5,6,9,10] → 6通道│
    │  BandSet 2: 通道[8,11] → 2通道        │
    └───────────────────────────────────────┘
         ↓
    ┌─────────────────────────────────────┐
    │  Patch Embedding (FlexiPatchEmbed)  │
    │  [H, W, 4] → [H/4, W/4, D=768]      │
    │  [H, W, 6] → [H/4, W/4, D=768]      │
    │  [H, W, 2] → [H/4, W/4, D=768]      │
    └─────────────────────────────────────┘
         ↓
输出: [B, H', W', T, S=3, D=768]
```

### 6. 模型推理

**源代码：** `olmoearth_inference_jp2_simple.py` 第347-354行

```python
# 构建模型（仅Encoder）
model = load_model_direct()
model.eval()

# 推理
with torch.no_grad():
    output = model(sample, fast_pass=True, patch_size=4)
    tokens_and_masks = output["tokens_and_masks"]
    
    # 获取sentinel2_l2a特征
    features = tokens_and_masks.sentinel2_l2a  # [B, H', W', T, S, D]
    
    # Pooled: 去掉 T 和 S 维度 -> [B, H', W', D]
    pooled = features.mean(dim=[3, 4])
```

**输出Shape汇总：**

| 张量 | Shape | 说明 |
|------|-------|------|
| `sentinel2_l2a` (输入) | `[B, H, W, T, C=12]` | 原始12波段数据 |
| `sentinel2_l2a_mask` | `[B, H, W, T, S=3]` | BandSet掩码 |
| `features` (输出) | `[B, H', W', T, S=3, D=768]` | 模型特征 |
| `pooled` | `[B, H', W', D=768]` | 时间+BandSet维度平均后的特征 |

其中 `H' = H / patch_size`, `W' = W / patch_size`

## 常见问题

### Q: 为什么有这个helios的东西？

A: 这是原项目内部的包命名，现在已经没有意义了。简化版脚本完全绕过了这个问题。

### Q: 为什么简化版不读config.json？

A: 因为完全没必要！模型参数是固定的（768维、12层），直接写死在代码里更简单。

### Q: weights.pth从哪下载？

A: https://huggingface.co/allenai/OlmoEarth-v1-Base

### Q: 推理结果是正确的吗？

A: 是的。简化版使用与原始完全相同的模型架构（nn/galileo.py, nn/flexi_vit.py），只是跳过了从配置文件加载参数这一步。

### Q: BandSet在数据流程中扮演什么角色？

A: BandSet定义了如何将原始波段分组。每个BandSet对应模型中的一个独立的token embedding路径。输入的12个连续波段被分成3组（4+6+2），每组独立通过 `FlexiPatchEmbed` 转换为embedding，最终输出 `[B, H', W', T, S=3, D]` 格式，其中S=3是BandSet的数量。

### Q: 归一化是在哪一步进行的？有哪几种策略？

A: 归一化在第3步进行，位于 `normalize_image()` 函数中。OlmoEarth支持两种归一化策略：

**策略1：PREDEFINED（使用预定义min/max）**
- 参数来源：`data/norm_configs/predefined.json`
- 公式：`(data - min) / (max - min)`
- 直接将数据线性缩放到[0,1]区间

**策略2：COMPUTED（使用mean/std计算，推荐）**
- 参数来源：`data/norm_configs/computed.json`
- 公式：`min = mean - 2*std`, `max = mean + 2*std`, `normalized = (data - min) / (max - min)`
- 将数据标准化到覆盖约95%数据分布的范围内

推理脚本默认使用COMPUTED策略。如需切换到PREDEFINED策略，需要完成以下两个步骤：

#### 步骤1：修改 `olmoearth_inference_jp2_simple.py`（切换策略）

**源代码：** `olmoearth_inference_jp2_simple.py` 第91行

```python
# 将 COMPUTED 改为 PREDEFINED
normalizer = Normalizer(strategy=Strategy.PREDEFINED)
```

#### 步骤2：修改/创建 `data/norm_configs/predefined.json`（添加/修改归一化参数）

**参数格式：** `data/normalize.py` 第78-95行

```python
modality_norm_values = self.norm_config[modality.name]  # e.g., "sentinel2_l2a"
min_val = modality_norm_values[band]["min"]   # e.g., 0
max_val = modality_norm_values[band]["max"]   # e.g., 10000
```

**JSON结构要求：**

```json
{
  "模态名称": {
    "波段名称": {
      "min": 数值,
      "max": 数值
    }
  }
}
```

**Sentinel-2 L2A的predefined.json示例**（已存在于 `data/norm_configs/predefined.json` 第620-673行）：

```json
{
  "sentinel2_l2a": {
    "B01": {"min": 0, "max": 10000},
    "B02": {"min": 0, "max": 10000},
    "B03": {"min": 0, "max": 10000},
    "B04": {"min": 0, "max": 10000},
    "B05": {"min": 0, "max": 10000},
    "B06": {"min": 0, "max": 10000},
    "B07": {"min": 0, "max": 10000},
    "B08": {"min": 0, "max": 10000},
    "B09": {"min": 0, "max": 10000},
    "B11": {"min": 0, "max": 10000},
    "B12": {"min": 0, "max": 10000},
    "B8A": {"min": 0, "max": 10000}
  }
}
```

**注意：** 波段名称必须与 `data/constants.py` 中定义的 `Modality.SENTINEL2_L2A.band_order` 一致：
```python
['B02', 'B03', 'B04', 'B08', 'B05', 'B06', 'B07', 'B8A', 'B11', 'B12', 'B01', 'B09']
```

**重要提示：** 使用PREDEFINED策略时，`predefined.json` 中必须包含所有使用的模态和波段。如果缺少某个波段，会抛出 `ValueError`（见 `data/normalize.py` 第88-89行）：
```python
if band not in modality_norm_values:
    raise ValueError(f"Band {band} not found in config")
```

### Q: 推理时有数据增强吗？

A: 没有。数据增强（如 `nn/flexi_vit.py` 中的 `band_dropout`）仅在训练时 `self.training=True` 时生效。推理时这些增强被跳过。
