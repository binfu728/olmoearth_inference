# OlmoEarth Inference 使用说明

## 快速开始

### 1. 下载模型权重

从HuggingFace下载OlmoEarth-v1-Base：
```bash
# https://huggingface.co/allenai/OlmoEarth-v1-Base
```

将`weights.pth`放入`params/`目录。

### 2. 安装依赖

```bash
pip install torch numpy rasterio einops
```

### 3. 运行推理

```bash
cd olmoearth_inference
python olmoearth_inference_jp2_simple.py --jp2_dir /path/to/sentinel2_safe
```

## 目录结构

```
olmoearth_inference/
├── olmoearth_inference_jp2_simple.py   # ⭐ 推荐使用这个脚本
├── olmoearth_inference_jp2.py           # 复杂版本（依赖config解析）
├── config.py                            # 配置解析（可选）
├── datatypes.py                         # 数据结构
├── types.py                            # 类型定义
├── nn/                                 # 神经网络模块
│   ├── galileo.py                      # 模型定义
│   ├── flexi_vit.py                    # Encoder/Decoder
│   └── ...（其他模块）
├── data/                               # 数据处理
│   ├── constants.py                     # 模态定义
│   ├── normalize.py                     # 归一化
│   └── norm_configs/                   # 归一化参数
└── params/                             # 模型权重
    └── weights.pth                     # ⭐ 从HuggingFace下载
```

## 两种脚本的区别

### olmoearth_inference_jp2_simple.py（推荐）

**优点：**
- 直接用代码定义模型参数，不依赖任何配置文件
- 简单粗暴，容易理解
- 不需要config.json

### olmoearth_inference_jp2.py（复杂版）

**缺点：**
- 需要正确的config.json
- 依赖复杂的config解析逻辑
- helios类路径映射容易出错

## 常见问题

### Q: 为什么有这个helios的东西？

A: 这是原项目内部的包命名，现在已经没有意义了。简化版脚本完全绕过了这个问题。

### Q: 为什么简化版不读config.json？

A: 因为完全没必要！模型参数是固定的（768维、12层），直接写死在代码里更简单。

### Q: weights.pth从哪下载？

A: https://huggingface.co/allenai/OlmoEarth-v1-Base

### Q: 推理结果是正确的吗？

A: 是的。简化版使用与原始完全相同的模型架构（nn/galileo.py, nn/flexi_vit.py），只是跳过了从配置文件加载参数这一步。
